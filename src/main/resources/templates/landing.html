Как это работает

От токена до первого инсайта — за пять минут

Вы не настраиваете дашборды. Вы получаете готовые ответы, как только синк закончился.

01 / token
Подключите GitLab
Read-only personal access token с правом read_api. Self-hosted и cloud — оба варианта работают.
⌁ ~30 сек
02 / sync
Выберите репозитории
Помечаете нужные проекты — мы тянем историю MR, commits, deploys и issues. Синк идёт в фоне, прогресс виден.
⌁ 3–5 мин
03 / insights
Получите ответы
Сразу видите: где bottleneck, у кого перегруз, какие MR застряли. Без настройки дашбордов и формул.
⌁ сразу

Безопасность

Ваш код — у вас. Мы видим только метаданные.

Никаких diff'ов и содержимого файлов. Только заголовки MR, авторы, ревью-реакции и таймстемпы.

Read-only токены
Только read_api. Мы технически не можем что-то изменить в репозитории.
AES-256 шифрование
Токены хранятся зашифрованными. Никогда не отдаются через API.
Audit log
Каждое действие в workspace логируется — видно кто, когда и что менял.
Self-hosted
Docker-контейнер для on-premise. Данные не выходят из вашего контура.

FAQ

Частые вопросы

01Что значит «tracked dev»?
Разработчик, по которому мы собираем метрики из GitLab — то есть автор хотя бы одного MR/коммита за период. Менеджеры, тимлиды и наблюдатели в платформе платно не считаются.
02Какие данные читает GitPulse?
Только метаданные через GitLab API: список проектов, MR, авторов, ревьюеров, метки времени, комментарии-метки. Содержимое файлов и diff'ы мы не запрашиваем и не храним.
03Можно подключить self-hosted GitLab?
Да. В настройках источников указываете URL вашего GitLab + токен. Поддерживаются и cloud (gitlab.com), и любые self-hosted инстансы.
04Как часто обновляются данные?
Каждые 30 минут на Team и Pro. На Free — раз в сутки. Можно запустить ручной sync в любой момент.
05А GitHub когда?
В работе. Если нужен GitHub — оставьте заявку на регистрацию, пришлём, как только будет готов.
06Можно отменить подписку?
В любой момент через настройки. Доступ остаётся до конца оплаченного периода. Никаких звонков и форм отказа.
